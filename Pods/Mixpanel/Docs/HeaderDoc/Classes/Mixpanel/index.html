Introduction

The primary interface for integrating Mixpanel with your app.

Discussion

Use the Mixpanel class to set up your project and track events in Mixpanel Engagement. It now also includes a people property for accesseing the Mixpanel People API. 

 // Initialize the API
 Mixpanel *mixpanel = [Mixpanel sharedInstanceWithToken:@"YOUR API TOKEN"];
 
 // Track an event in Mixpanel Engagement
 [mixpanel track:@"Button Clicked"];
 
 // Set properties on a user in Mixpanel People
 [mixpanel.people identify:@"CURRENT USER DISTINCT ID"];
 [mixpanel.people set:@"Plan" to:@"Premium"];

For more advanced usage, please see the Mixpanel iPhone Library Guide.

archive

- (void)archive;
Discussion

This state will be recovered when the app is launched again if the Mixpanel library is initialized with the same project token. You do not need to call this method. The library listens for app state changes and handles persisting data as needed. It can be useful in some special circumstances, though, for example, if you'd like to track app crashes from main.m.

Discussion

By default, queued data is flushed to the Mixpanel servers every minute (the default for flushInvterval), and on background (since flushOnBackground is on by default). You only need to call this method manually if you want to force a flush at a particular moment.

Discussion

Super properties, once registered, are automatically sent as properties for all event tracking calls. They save you having to maintain and add a common set of properties to your events. Property keys must be NSString objects and values must be NSString, NSNumber, NSNull, NSArray, NSDictionary, NSDate or NSURL objects.

sharedInstanceWithToken:

Initializes and returns a singleton instance of the API. 

+ (id)sharedInstanceWithToken:(NSString *)apiToken;
Parameters
apiToken

your project token

Discussion

If you are only going to send data to a single Mixpanel project from your app, as is the common case, then this is the easiest way to use the API. This method will set up a singleton instance of the Mixpanel class for you using the given project token. When you want to make calls to Mixpanel elsewhere in your code, you can use sharedInstance. 

 [Mixpanel sharedInstance] track:@"Something Happened"]];

If you are going to use this singleton approach, sharedInstanceWithToken: must be the first call to the Mixpanel class, since it performs important initializations to the API.

Discussion

A distinct ID is a string that uniquely identifies one of your users. Typically, this is the user ID from your database. By default, we'll use a hash of the MAC address of the device.
